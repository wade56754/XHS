## 标题硬性约束（必须全部满足）

### MUST DO
1. **必须有数字**：具体数字（3 个、4 小时→20 分钟）或对比（之前 vs 现在）
2. **必须有场景**：谁、在什么情况下、解决什么问题
3. **必须第一人称或疑问句**：「我用...」「你是不是也...」「为什么我...」
4. **必须 18-25 字**：最长不超过 28 字，超过的砍掉

### MUST NOT
1. **禁止知乎问题式**：「如何看待 X」「X 和 Y 的区别是什么」
2. **禁止技术 blog 式**：「基于 X 的 Y 实现」「从 0 到 1 搭建」
3. **禁止 PPT 标题式**：「X 的多种应用场景探索」「X 系统设计思路分享」
4. **禁止信息堆砌**：一个标题只传达 1 个核心卖点 + 1 个钩子

## 标题公式参考（请模仿这些结构）

| 公式 | 示例 |
|------|------|
| 数字 + 结果 + 身份 | 我用 3 个自动化流程，把每天 4 小时重复工作干没了 |

| 数字 + 对号入座 | 我实践过的 8 种可自动化工作，哪种你天天在做？ |
