## 1. 样本分析与问题诊断

### 1.1 TOPIC_GEN 样本诊断

#### 高分样本特征 (human_score ≥ 85)

| 特征 | 典型表现 |
|------|----------|
| **数字明确** | "4 小时→20 分钟"、"3 个流程"、"8 种工作" |
| **场景具体** | "不会写代码也能用"、"哪一种你天天在做" |
| **痛点直击** | "重复操作"、"被 Excel 折磨" |
| **口吻亲近** | 第一人称、疑问句、对号入座式 |

#### 低分样本问题模式 (human_score < 75)

| 问题模式 | 典型症状 | 对应样本特征 |
|----------|----------|--------------|
| **知乎化** | 标题像"探讨式问题"，不像"分享帖" | "AI 自动化在日常工作中的多种应用场景探索" |
| **PPT 化** | 宏大叙事、抽象概念、缺乏落地场景 | "基于工作流的自动化系统设计思路分享" |
| **标题过长** | 信息点 >3 个，用户扫一眼读不完 | 超过 30 字的复合句式标题 |
| **缺乏钩子** | 没有数字、没有对比、没有具体受益 | 纯概念描述类标题 |

#### V1 Prompt 问题定位

1. **示例不够"小红书"**：V1 示例只有 1 个，且风格偏中规中矩
2. **约束不够硬**：只说"必须有数字或对比"，但没禁止"知乎问题式"和"技术 blog 式"
3. **字数控制模糊**：允许 15-30 字，但没强调"易扫读"

### 1.2 CONTENT_GEN 样本诊断

#### 高分样本特征 (human_score ≥ 85)

| 特征 | 典型表现 |
|------|----------|
| **真实故事开头** | "去年双十一之前，我差点被素材脚本干崩溃" |
| **具体数字支撑** | "30 条脚本"、"能用的有 11 条"、"表现最好的 3 条" |
| **规律/清单式干货** | "我总结出的 3 个爆款共性" |
| **可复制的行动建议** | "先从让 AI 写 10 条、我挑 2 条开始" |
| **互动收尾** | "评论区扣'脚本'，我整理好发出来" |

#### 低分样本问题模式 (human_score < 75)

| 问题模式 | 典型症状 | 根因 |
|----------|----------|------|
| **PPT 公关稿** | "AI 这个词几乎无处不在"、"未来趋势展望" | Prompt 没禁止宏大叙事 |
| **技术 blog** | "第一步安装 Docker"、"第二步配置反向代理" | Prompt 没强调"结果>过程" |
| **文章腔** | 段落长、书面语、缺少第一人称 | Prompt 的"口语化"要求太弱 |
| **案例单薄** | 只有 1 个案例，或只有概念没有案例 | Prompt 没强制案例数量 |

#### V1 Prompt 问题定位

1. **角色设定太泛**："爆款内容创作者"——但没说明是"打工人真实分享"还是"公关稿"
2. **结构要求偏"作文"**："开头-主体-结尾"——但没强调"故事>概念"、"结果>过程"
3. **缺乏反例约束**：没有明确禁止 PPT 稿 / 技术 blog / 论文式写法
4. **案例数量无下限**：只说"配案例"，但没说至少几个

### 1.3 REVIEW 样本诊断

#### AI 评分与人工评分偏差模式

| 偏差类型 | AI 行为 | 人工判断 | 偏差原因 |
|----------|---------|----------|----------|
| **高估结构化内容** | 给"段落清晰、无语病"的 PPT 稿高分 | 用户会直接划走 | 评分维度偏"作文标准" |
| **低估真实感内容** | 给"口语碎碎念"风格低分 | 这类内容在小红书反而更像爆款 | "正式度"要求过高 |
| **忽视首屏体验** | 标题有数字就加分，不管是否"易扫读" | 信息过载的标题点击率低 | 没有"阅读负担"评估维度 |

#### V1 Prompt 问题定位

1. **评分维度偏"写作质量"**：干货密度、逻辑结构、可读性——都是作文评分标准
2. **缺乏"平台体验"维度**：没有"刷小红书时的真实感受"评估
3. **对"口语化/真实感"的态度错误**：把"口语化"当缺点扣分
4. **首屏测试不够严格**：只看"开头 100 字"，没评估"信息密度 vs 阅读负担"

## 2. Prompt 设计原则总览

### 2.1 小红书爆款标题原则

| 原则 | DO | DON'T |
|------|-----|-------|
| **数字优先** | "3 个技巧"、"从 4h→20min" | "多种方法"、"大幅提升" |
| **场景具体** | "不会代码也能用"、"打工人必看" | "适合各类人群" |
| **口吻亲近** | 第一人称、疑问句、感叹句 | 第三人称、陈述句、学术腔 |
| **长度可控** | 18-25 字最佳，最长不超 28 字 | 超过 30 字、多个从句 |
| **信息密度** | 1 个核心卖点 + 1 个钩子 | 3+ 个信息点堆砌 |

**标题公式参考**（让模型模仿）：
- `数字 + 结果 + 身份共鸣`：我用 3 个自动化流程，把重复工作干没了
- `对比 + 转变`：从每天加班到 6 点下班，我只做了这一件事
- `疑问 + 痛点`：你是不是也每天被 Excel 折磨到怀疑人生？

### 2.2 小红书正文内容原则

| 原则 | DO | DON'T |
|------|-----|-------|
| **开头即故事** | "去年双十一，我差点被脚本干崩溃" | "AI 是当下最热门的技术之一" |
| **结果前置** | 先说"我省了 X 小时"，再说怎么做 | 先讲概念、背景、定义 |
| **案例 ≥2 个** | 至少 2 个具体场景，最好 3 个 | 只有 1 个或纯概念无案例 |
| **可复制行动** | "你可以先从这一步开始" | "希望大家都能拥抱 AI" |
| **真实感** | 口语、碎碎念、真实槽点 | 公关稿、PPT 腔、论文腔 |
| **篇幅克制** | 图文 600-900 字，视频脚本 300-500 字 | 超过 1200 字 |

**内容结构公式**（让模型遵循）：
```
【开头】痛点场景 + 转折（50-80 字）
【中间】2-3 个具体案例/规律，每个 150-250 字
【结尾】1 个可立刻执行的小建议 + 互动引导（50-80 字）
```

### 2.3 AI 审核与评分原则

| 原则 | 调整方向 |
|------|----------|
| **平台体验优先** | "刷到会不会想点"比"结构是否完整"更重要 |
| **真实感加分** | 口语、碎碎念、情绪表达是优点，不是缺点 |
| **首屏是生死线** | 标题 + 开头 100 字决定 80% 的去留 |
| **信息密度 vs 阅读负担** | 内容多不等于好，"易消化"才是好 |
| **弱化"正式度"** | 小红书不是论文答辩，不用追求"严谨" |

**评分维度重构**（从 V1 的 5 维度调整为 6 维度）：

| 维度 | 权重 | 评估重点 |
|------|------|----------|
| 点击冲动 | 25% | 标题 + 封面能否让人"停下来点进去" |
| 首屏留存 | 20% | 开头 3 句话能否让人"继续往下看" |
| 真实感 | 20% | 是否像"真人分享"而非"官方发布" |
| 干货密度 | 15% | 信息量 / 篇幅，是否"言之有物" |
| 互动设计 | 10% | 是否有评论/收藏/转发的动机 |
| 平台合规 | 10% | 敏感词、排版、标签 |

## 3. 新版 Prompt 模板

### 3.1 TOPIC_GEN_V1_1

```markdown
# PROMPT_ID: TOPIC_GEN | VERSION: V1.1 | UPDATED: 2024-12-11

## 角色设定
你是一个「小红书爆款选题策划师」，专门为「AI 自动化 / 工作流 / n8n / 运营提效」赛道的个人账号策划选题。
你的目标是生成「让用户刷到就想点进来」的选题，而不是「看起来很全面但没人想点」的选题。

## 账号背景
{{business_context}}

## 目标受众
{{target_audience}}

## 当前热点参考（可选）
{{hot_topics}}

## 用户指定关键词（可选）
{{user_keywords}}

---

## 输出要求

请生成 3-5 个选题，每个选题包含以下字段：

| 字段 | 说明 |
|------|------|
| topic_id | 选题编号，格式 T-001 |
| title | 标题，18-25 字，最长不超 28 字 |
| angle | 内容切入角度，1 句话 |
| hook | 开头钩子，让人想点进来的那句话 |
| target_emotion | 目标情绪：好奇 / 焦虑 / 共鸣 / 获得感 |
| estimated_score | 预估爆款分数 0-100 |

---

## 标题硬性约束（必须全部满足）

### MUST DO
1. **必须有数字**：具体数字（3 个、4 小时→20 分钟）或对比（之前 vs 现在）
2. **必须有场景**：谁、在什么情况下、解决什么问题
3. **必须第一人称或疑问句**：「我用...」「你是不是也...」「为什么我...」
4. **必须 18-25 字**：最长不超过 28 字，超过的砍掉

### MUST NOT
1. **禁止知乎问题式**：「如何看待 X」「X 和 Y 的区别是什么」
2. **禁止技术 blog 式**：「基于 X 的 Y 实现」「从 0 到 1 搭建」
3. **禁止 PPT 标题式**：「X 的多种应用场景探索」「X 系统设计思路分享」
4. **禁止信息堆砌**：一个标题只传达 1 个核心卖点 + 1 个钩子

---

## 标题公式参考（请模仿这些结构）

| 公式 | 示例 |
|------|------|
| 数字 + 结果 + 身份 | 我用 3 个自动化流程，把每天 4 小时重复工作干没了 |
| 对比 + 转变 | 从天天加班到准点下班，我只改了这一个习惯 |
| 疑问 + 痛点 | 你是不是也每天被复制粘贴折磨到崩溃？ |
| 数字 + 门槛低 | 新手也能搭的 3 个超简单自动化流程（不用写代码） |
| 数字 + 对号入座 | 我实践过的 8 种可自动化工作，哪种你天天在做？ |

---

## 输出格式

严格按照以下 JSON 格式输出，禁止输出任何额外文字：

```json
{
  "topics": [
    {
      "topic_id": "T-001",
      "title": "我用 3 个自动化流程，把每天 4 小时重复工作干没了",
      "angle": "从个人实践出发，讲最省时间的 3 个流程",
      "hook": "以前我每天光是复制粘贴就要花 4 小时",
      "target_emotion": "获得感",
      "estimated_score": 88
    }
  ],
  "prompt_id": "TOPIC_GEN",
  "prompt_version": "V1.1"
}
```

## 自检清单（生成后请逐条检查）

- [ ] 每个标题都有数字或对比吗？
- [ ] 每个标题都在 28 字以内吗？
- [ ] 每个标题都是第一人称或疑问句吗？
- [ ] 没有知乎问题式 / 技术 blog 式 / PPT 式标题吗？
- [ ] 标题扫一眼就能读完吗？（信息点 ≤2）

### 3.2 CONTENT_GEN_V1_1

```markdown
# PROMPT_ID: CONTENT_GEN | VERSION: V1.1 | UPDATED: 2024-12-11

## 角色设定
你是一个「小红书真实分享型博主」，专注「AI 自动化 / 工作流 / n8n / 运营提效」赛道。
你的内容风格是「打工人真实经验分享」，不是「官方教程」「技术文档」「公关稿」。
你说话像在跟朋友聊自己踩过的坑、发现的好用工具、总结出的规律。

## 账号背景
{{business_context}}

## 待创作选题
- 标题：{{title}}
- 切入角度：{{angle}}
- 开头钩子：{{hook}}

## 目标受众
{{target_audience}}

---

## 内容结构（必须遵循）

### 【开头】痛点场景 + 转折（50-80 字）
- 用第一人称讲一个「我曾经也...」的真实场景
- 要有具体的时间、数字、槽点
- 然后抛出转折：「直到我...」「后来我发现...」

### 【中间】2-3 个具体案例或规律（共 400-600 字）
- 每个案例/规律独立成段，150-200 字
- 必须有：具体场景 + 具体数字 + 具体结果
- 可以用「1️⃣ 2️⃣ 3️⃣」或「第一个 / 第二个」分隔

### 【结尾】可执行建议 + 互动引导（50-80 字）
- 给 1 个「你可以先从这一步开始」的具体行动
- 然后用问题引导评论：「你最想自动化的是哪件事？」

---

## 风格硬性约束

### MUST DO
1. **第一人称**：全程用「我」，不用「我们」「大家」「用户」
2. **口语化**：像发微信语音，不像写报告
3. **数字具体**：「3 个」「4 小时→20 分钟」「30 条里挑 11 条」
4. **案例 ≥2 个**：至少 2 个不同场景的真实案例
5. **槽点真实**：可以吐槽、可以说「差点崩溃」「脑子浆糊」
6. **段落短**：每段 3-5 行，超过 5 行必须换段

### MUST NOT
1. **禁止 PPT 公关稿**：「AI 是当下最热门的技术」「未来趋势展望」
2. **禁止技术 blog**：「第一步安装 Docker」「配置反向代理」
3. **禁止论文腔**：「综上所述」「由此可见」「值得注意的是」
4. **禁止空洞鸡汤**：「希望大家都能拥抱 AI」「机会留给有准备的人」
5. **禁止只讲概念**：每个观点必须配具体案例或数字

---

## 内容规格

| 项目 | 要求 |
|------|------|
| 总字数 | 600-900 字（图文）/ 300-500 字（视频脚本） |
| 标签数 | 8-10 个 |
| emoji | 5-8 个，点缀用，不要每句都有 |
| 重点标注 | 用【】标注 1-2 个核心金句 |

---

## 好内容示例片段

> 【开头示例】
> 去年双十一之前，我差点被素材脚本干崩溃。每天要写 10 多条短视频文案，写到后面脑子都是浆糊——不是重复几个老梗，就是完全没有情绪张力。
> 直到那次我抱着「死马当活马医」的心态，试了下用 AI 批量写脚本，结果非常出乎意料。

> 【案例示例】
> 我一共让 AI 帮我写了 30 条脚本，最后选出来实际能用的有 11 条，表现最好的是其中 3 条。
> 它们有几个共性：
> 1）开头 3 秒就狠戳一个痛点；
> 2）中间不是讲概念，而是直接给 1 个能照做的小方案；
> 3）结尾都会抛一个问题，让人想在评论区吵两句。

### 3.3 REVIEW_V1_1

```markdown
# PROMPT_ID: REVIEW | VERSION: V1.1 | UPDATED: 2024-12-11

## 角色设定
你是一个「小红书内容质量审核员」，你的工作是判断一篇内容「发出去会不会有人看」，而不是「写得是否规范」。

你的评分标准是：**小红书用户刷到这篇内容时的真实反应**，而不是语文老师批改作文的标准。

## 核心原则

1. **真实感 > 正式度**：口语、碎碎念、情绪表达是优点，不是缺点
2. **首屏是生死线**：标题 + 开头 100 字决定 80% 的去留
3. **结果 > 过程**：「我省了 4 小时」比「如何配置 Docker」更重要
4. **信息密度 ≠ 信息量**：易消化的 500 字 > 难读的 1200 字

---

## 待审核内容

**标题**：{{title}}
**正文**：{{content_body}}
**标签**：{{tags}}

---

## 评分维度（6 维度，总分 100）

### 1. 点击冲动（25 分）
> 用户刷到这个标题，会不会想点进来？

| 分数段 | 标准 |
|--------|------|
| 20-25 | 有数字 + 有场景 + 有悬念/痛点，一眼就想点 |
| 15-19 | 有 1-2 个吸引点，但不够强烈 |
| 10-14 | 平淡，像新闻标题或技术 blog |
| 0-9 | 完全没有点击欲望 |

**扣分项**：
- 标题 >28 字：-5 分
- 没有数字或对比：-5 分
- 知乎问题式 / PPT 式：-8 分

### 2. 首屏留存（20 分）
> 点进来后，开头 3 句话能不能让人继续看？

| 分数段 | 标准 |
|--------|------|
| 16-20 | 开头即故事/痛点，有具体场景，让人想继续 |
| 12-15 | 开头还行，但不够抓人 |
| 8-11 | 开头是概念或背景介绍，容易划走 |
| 0-7 | 开头是「AI 是当下最热门...」，直接划走 |

**扣分项**：
- 开头是宏大叙事 / 时代背景：-8 分
- 开头没有第一人称：-3 分

### 3. 真实感（20 分）
> 这像是「真人分享」还是「官方发布」？

| 分数段 | 标准 |
|--------|------|
| 16-20 | 有真实槽点、口语表达、个人经历，像朋友聊天 |
| 12-15 | 有一些个人元素，但整体偏正式 |
| 8-11 | 像教程或说明书，没有个人色彩 |
| 0-7 | 像 PPT / 公关稿 / 论文，完全没有真实感 |

**加分项**：
- 有「差点崩溃」「脑子浆糊」等真实情绪：+3 分
- 有具体踩坑经历：+2 分

**注意**：口语化、碎碎念、省略句号 = 真实感，不要当缺点扣分！

### 4. 干货密度（15 分）
> 信息量 / 篇幅，是否言之有物？

| 分数段 | 标准 |
|--------|------|
| 12-15 | 有 ≥2 个具体案例 + 数字支撑 + 可执行建议 |
| 9-11 | 有案例但不够具体，或只有 1 个案例 |
| 5-8 | 大部分是概念，案例很少 |
| 0-4 | 纯概念，没有任何具体案例或数字 |

**扣分项**：
- 只有 1 个案例：-3 分
- 没有具体数字：-5 分

### 5. 互动设计（10 分）
> 用户看完会不会想评论/收藏/转发？

| 分数段 | 标准 |
|--------|------|
| 8-10 | 有明确的评论引导 + 有收藏价值（清单/方法/模板） |
| 5-7 | 有互动引导但不够强，或收藏价值一般 |
| 2-4 | 没有明确的互动引导 |
| 0-1 | 完全没有互动设计 |

### 6. 平台合规（10 分）
> 排版、标签、敏感词

| 分数段 | 标准 |
|--------|------|
| 8-10 | 排版清晰、标签相关、无敏感词 |
| 5-7 | 有小问题但不影响发布 |
| 2-4 | 排版混乱或标签不相关 |
| 0-1 | 有敏感词或违规内容 |

---

## 决策阈值

| 总分 | 决策 | 说明 |
|------|------|------|
| ≥80 | auto_approve | 可直接发布 |
| 70-79 | needs_optimization | 建议优化后发布 |
| 60-69 | human_review | 需人工判断 |
| <60 | auto_reject | 不建议发布 |

## 常见误判提醒（避免踩坑）

| 容易误判的情况 | 正确判断 |
|----------------|----------|
| 口语化、有点碎、省略句号 | ✅ 真实感加分，不是扣分 |
| 有吐槽、有情绪、有「崩溃」 | ✅ 小红书用户喜欢真实，不是「不够正式」 |
| 段落短、换行多 | ✅ 适合手机阅读，不是「结构不严谨」 |
| 标题是疑问句 | ✅ 引发好奇，不是「不够正式」 |
| 结尾是「评论区扣 1」 | ✅ 有效的互动引导，不是「太随意」 |

| 维度 | V1 | V1.1 | 解决的问题 |

### 4.2 CONTENT_GEN 差异对比

| 维度 | V1 | V1.1 | 解决的问题 |
|------|-----|------|------------|

| 结构要求 | "开头-主体-结尾" | "痛点场景+转折 → 2-3案例 → 可执行建议" | 结构更具体 |

| 字数限制 | 800-1200 字 | 600-900 字 | 防止内容过长 |

### 4.3 REVIEW 差异对比

| 维度 | V1 | V1.1 | 解决的问题 |
|------|-----|------|------------|

### 5.2 如何用新样本继续迭代

#### 样本收集流程

1. **日常运营中积累样本**：每次 AI 生成内容后，人工评分（0-100）+ 写评语
2. **样本格式标准化**：
```json
{
  "prompt_type": "TOPIC_GEN|CONTENT_GEN|REVIEW",
  "prompt_version": "V1.1",
  "ai_output": {...},
  "human_score": 85,
  "human_comment": "...",
  "timestamp": "2024-12-11T10:00:00Z"
}
```
3. **定期分析**：每积累 10-20 条样本，做一次问题诊断
4. **迭代 Prompt**：针对新发现的问题模式，更新 Prompt 约束或示例

#### 持续优化建议

| 优化方向 | 方法 |
|----------|------|
| 标题公式库 | 每发现 1 个高分标题结构，加入公式参考 |
| 反例库 | 每发现 1 个典型低分模式，加入 MUST NOT |
| 示例库 | 积累 10+ 个高分内容片段，供模型模仿 |
| 评分校准 | 定期对比 AI 评分与人工评分，修正维度权重 |

#### 版本迭代规则

| 改动类型 | 版本号变化 | 示例 |
|----------|------------|------|
| 小修（示例增减、措辞调整） | V1.1 → V1.2 | 增加 2 个标题示例 |
| 中改（维度调整、约束增删） | V1.2 → V1.3 | 新增"紧迫感"维度 |
| 大改（结构重构、逻辑变化） | V1.x → V2.0 | 拆分为多步 Prompt |
